What's broken right now

  1. No facility API. FacilityRepository in the Flutter app calls /api/facility/orders, /scan, /stats, /status — FacilityEndpoints.cs does not exist. Every facility screen 404s on load.
  2. Status vocabulary mismatch. Facility UI expects AtFacility / Washing / Ironing / Ready. API enum has InProcess / ReadyForDelivery. Even after endpoints exist, nothing will render correctly until one side is renamed.
  3. No "drop at facility" step. After confirm-pickup, the rider's assignment is Completed and the order sits at PickedUp forever. Nobody hands it to the facility.
  4. No delivery dispatcher. When an order reaches ReadyForDelivery, no code creates a delivery OrderAssignment. Riders never see delivery jobs on the /me/jobs endpoint.
  5. No customer-facing handoff notifications. SignalR pushes exist for rider assignment but not for "at facility" or "ready for delivery" transitions.
